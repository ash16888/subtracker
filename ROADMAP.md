# План улучшений SubTracker

## Цель

Сделать SubTracker более надежным инструментом управления подписками: с понятными расчетами, устойчивыми уведомлениями, удобным управлением подписками и практичной аналитикой расходов.

## Этап 0. Исправить основу

- Привести финансовые расчеты к единой логике во всех местах интерфейса: шапка, дашборд, список подписок, инсайты.
- Разделить показатели "расходы в этом месяце", "среднее в месяц" и "прогноз на 12 месяцев", чтобы пользователь не видел противоречивые суммы.
- Проверить расчет следующих платежей, особенно для старых, просроченных и ежегодных подписок.
- Добавить статусы подписок: активна, пробный период, приостановлена, отменена, архив.
- Показывать статус синхронизации с Google Calendar по каждой подписке.
- Добавить журнал ошибок синхронизации и кнопку повторной попытки.

## Этап 1. Улучшить управление подписками

- Переработать форму добавления и редактирования подписки в более компактный модальный экран или side panel.
- Добавить поиск по названию сервиса.
- Добавить сортировки: по дате платежа, сумме, категории, названию.
- Добавить комбинированные фильтры: категория, период оплаты, статус, наличие ссылки управления.
- Поддержать произвольные периоды оплаты: раз в 2 недели, раз в 3 месяца, раз в 6 месяцев, произвольное число дней.
- Добавить дату начала подписки, дату отмены, признак автопродления.
- Добавить архивирование без удаления истории.
- Добавить массовые действия: удалить, архивировать, сменить категорию, включить или выключить напоминания.
- Добавить дублирование подписки как быстрый способ создать похожую запись.

## Этап 2. Усилить финансовую аналитику

- Разделить фактические платежи, прогнозные платежи и усредненные месячные расходы.
- Добавить график расходов по месяцам.
- Показывать изменение расходов месяц к месяцу и год к году.
- Добавить бюджеты по категориям и общий месячный лимит.
- Показывать предупреждения при приближении к бюджету.
- Добавить поддержку нескольких валют с курсом конвертации.
- Показывать стоимость подписки в день, месяц и год.
- Находить дорогие, редко используемые или дублирующиеся подписки.
- Показывать потенциальную экономию при отмене или переходе на годовой тариф.

## Этап 3. Улучшить платежи и уведомления

- Добавить настройки напоминаний для каждой подписки: за 1, 3, 7 или 14 дней.
- Поддержать разные каналы уведомлений: Google Calendar, email, push, Telegram.
- Добавить отдельные напоминания об окончании пробного периода.
- Сделать календарный вид платежей на месяц.
- Добавить действие "платеж оплачен" с сохранением истории.
- Показывать просроченные и пропущенные платежи.
- Добавить историю платежей по каждой подписке.

## Этап 4. Добавить автоматизацию

- Добавить импорт подписок из CSV.
- Добавить импорт из банковской выписки, если пользователь загружает файл вручную.
- Находить возможные дубликаты подписок и предлагать объединение.
- Автоматически подбирать категорию, логотип и цвет сервиса.
- Подставлять официальную ссылку управления подпиской для популярных сервисов.
- Добавить шаблоны популярных сервисов: Netflix, Spotify, YouTube Premium, ChatGPT, iCloud, Google One и другие.

## Этап 5. Сделать AI-инсайты практичными

- Перевести AI-инсайты из текстового отчета в набор карточек с действиями.
- Для каждого инсайта показывать причину, ожидаемую экономию и уровень уверенности.
- Добавить сценарии: "как сэкономить", "что отменить", "уложиться в бюджет", "найти дубликаты".
- Позволить применять рекомендации прямо из карточки, например архивировать подписку или изменить период.
- Сравнивать месячный и годовой тариф, если это применимо.
- Не предлагать покупать новые сервисы, если пользователь явно не просит рекомендации.

## Этап 6. Улучшить дизайн и UX

- Уменьшить визуальную перегрузку дашборда: оставить самые важные показатели выше остальных.
- Сделать главными метриками месячные расходы, ближайший платеж и потенциальную экономию.
- Использовать цветовые карточки точечно, а не для каждой вторичной метрики.
- Сделать список подписок более сканируемым: логотип, название, сумма, дата платежа, статус, быстрые действия.
- Добавить компактный режим списка.
- Добавить темную тему.
- Улучшить мобильную версию: нижняя навигация, удобные карточки, форма без горизонтального сжатия.
- Добавить skeleton/loading, empty state и error state.
- Улучшить доступность: aria-label для иконок, фокус-состояния, контраст, управление с клавиатуры.

## Этап 7. Расширить продукт

- Добавить семейные или общие подписки.
- Добавить распределение стоимости между участниками.
- Добавить PWA-режим и базовую работу офлайн.
- Добавить экспорт в CSV и PDF.
- Добавить резервное копирование и восстановление данных.
- Добавить раздел приватности: экспорт данных, удаление аккаунта, очистка календарных событий.

## Приоритет внедрения

1. Исправить расчеты, даты и статусы подписок.
2. Добавить историю платежей и устойчивую синхронизацию с календарем.
3. Улучшить форму, список, поиск, фильтры и сортировку.
4. Добавить уведомления и календарный вид платежей.
5. Добавить бюджеты и более точную аналитику.
6. Добавить импорт, шаблоны сервисов и автоматическое обогащение данных.
7. Переработать AI-инсайты в практичные рекомендации.
8. Добавить семейные функции, экспорт, PWA и расширенные настройки приватности.
